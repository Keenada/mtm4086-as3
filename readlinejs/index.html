Introduction To Readline.js

Readline.js is a core module included in node.js.

Readline is a module that allows reading of a stream on a line-by-line basis. So what does that mean? That means when readline is running, you can track and store inputs from users. How is this usefull? Well, git ready, because we're about to drop some node-ledge on you.

How to use Readline.js

To use readline.js, you would need to include it in your file.

const readline = require('readline');

Then you have to setup a simple input interface. The interface gives users a place to input values.

const rl = readline.createInterface(process.stdin, process.stdout);

If you don't close the interface, it will continue to run. It needs to be told when to close. To close the interface instance, you need to use the 'close' event. The syntax looks like this:

rl.close();

Example 1

You can use readline.js to callback input values from users. Here is the syntax:

rl.question("what is favourite your colour?", (answer)

The prompt is prepended with a query and invokes a callback with the user's response. It displays the query to the user, and then invokes callback with the user's response after it has been typed and entered.

Example 3

Now, let's take a look at doing something useful. You can setup a promt and let the user type in whatever they want. You can setup readline.js to wait for the user to type in a specific input and respond accordingly.

It's kind of like those old text based commodore 64 games, if you type in the right input, something will happen.

Readline is waiting for 2 different inputs. If the user types in "hello", readline will output "OLA!". If the user types in "close", readline outputs "Have a gread day!" and then it will close itself. Any other input will have a default responce of "Sorry, I dont know what (user input) means".
