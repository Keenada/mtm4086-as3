What is EventEmitter?

The event emitter is a class inside Node core used to create applications based off events. An event handler is triggered by an event listener which is waiting for an event to occur such as a “click”, “tap”, or “timer-off”. The event emitter function is fundamental to almost any dynamic application where multiple things can be happening simultaneously.

The way Node (or JavaScript language) works is that it’s almost always based off events that occur which cause the application to flow further along. For example: it can be thought of as a story; an application must have a start and end but events happen in between which progress the story forward, before finally arriving to the end scene. For ex: an event listener can be “buzz, buzz, buzz” (it’s a doorbell). The application has been listening for this event so that an action can be completed once someone rings the doorbell. The application sets of another event handler which triggers the camera to “on” in order to visually identify who is at the front door by displaying a live feed inside a web application to a home owner using an iPad. Node is also a bit different (slightly more advanced) than JavaScript since you are able to emit events without having a user interact with the application and are able to trigger events without any user interaction.

Let's Dive In!

In order to start using the eventEmitter class you must include the “Events” module.

const StartUsingEventEmitter = require('events');

Here we will go through an example of the EventEmitter class being used to trigger an event once a timer has completed. In order to begin using the methods associated with the EventEmitter class we need to set a new “event emitter” object which we call myCookingTimer.

var MyEventEmitter = require("events").EventEmitter;
var myCookingTimer = new EventEmitter();

Here we are listening for the event “SwitchOvenOff” to be fired by some other part of the application. This function is using one of the two main methods part of the event emitter class. One being the “on” method which is now actively listening for the “SwitchOvenOff” event.

This function takes two parameters the first one is the event listener which we can call anything like previously stated this could be a “click”, “tap” etc.. in our case: switch the oven off event. We also have a second parameter which is the name of the event handler or function that will run after an event has occurred.

myCookingTimer.on('SwitchOvenOff', function(TodaysDinner) {
alert(“Dinner is Served we are having: ” + TodaysDinner );
OvenOff();
});

Putting it all together

Once the event listener has been initialized properly, we set-off the timer and are waiting for the timer to complete. After about 30 minutes as in our case, the “emit” method will fire off an event with the name of whatever value we have defined as our first parameter (“SwitchOvenOff”). As you notice this is the same name we have set in the previous step, that is so our event listener knows what to listen for, it may be true that you have multiple events inside an application. Thus, preventing all the different events to go off at the same time. In the second field we can add an additional parameter, since we have decided to cook “chicken” we are passing a variable back to the function so that we can do additional things with that information.

setInterval ( function(chicken) {
myCookingTimer.emit ('SwitchOvenOff', chicken);
}, 45000);

This also brings us to another method of the Event Emitter class. That is the “once” method. It is called once and only once. It will prevent any events in the future from doing their job pertaining to that particular event. Node knows to only call that event once and any future events of it will not initiate a call back function.

myEventEmitter.once("SomethingHasHappend", function () {
alert("I'm only seen once!");
});

There may also be a case where you want to remove an event listener. We have a method called “removeListener”. We use it the same was ay we would when we set an event listener with two parameters. Once again the first parameter holds the name/kind of “event” we want to stop listening for and the second parameter is the callback function.
