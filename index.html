Introduction:

A library for node, Pdfkit allows the use of server-side generation of complex, multi-page, printable pdf documents. The API can be used in javascript or CoffeeScript, to varying degrees of success (which you will witness throughout this tutorial).

The image left will take you to an in-browser, interactive pdf generator.

Getting Started:

To get started you must first install pdfkit in node by typing the code snippet to the right into npm. Once you have installed the module, create a file to work from.

I will call mine thisFile.js

node

npm install pdfkit

Adding Text:

Pdfkit makes provision for adding and styleing text in a number of ways. The best way to add text is to create a string variable in which your text is housed. You then inject that variable into your pdf with the following code.

Assuming this code worked right you would get a paragraph outputted, over a 3 column span.

--step2

//put text in a variable
var text = 'Cookie jelly beans chocolate chocolate soufflé cookie carrot cake sugar plum pie. Sesame snaps macaroon cookie jujubes macaroon pie jelly beans bonbon candy. Ice cream halvah topping muffin chocolate bar bonbon tiramisu chocolate cake wafer. Jujubes cupcake toffee dragée liquorice pastry gingerbread. ';

doc.text(text, {
//style variable text
columns: 3 //set columns
columnGap: 20, //set column margin
height: 500, //set hight of columns
width: 300, //set width of columns
align: 'left' //set alignment of columns
});

--step3

doc.poly()
     ..moveTo(50, 100) //specify line segments for polygon
     ..lineTo(140, 450)
     ..lineTo(150, 2150)
     ..fill("#B0EAAD"); //fill color

Adding Graphics:

Pdfkit allows the creation of shapes and graphics (similar to css shapes) through the input of various presets. You can also create your own shapes by declaring there vertices and line segments, and then their fill color...at least in theory.

Adding Images:

Adding images is very easy to do in pdfKit (according to the documentation). Just a few simple lines of code and ideally, you will have set an image that can be resized extremely simply...ideally...probably...

--step4

doc.image('images/image.jpeg', 0, 15, width: 500) //insert image path, position and specify width
     .text('Proprotional to width', 0, 0) //scale image
