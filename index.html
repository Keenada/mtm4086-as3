What is Prune?

So you want to understand Prune.js. Prune is a core node module that allows you to remove extraneous modules that are not included in a package.json file. An extraneous module is a module that is not included as a dependency or devDependency in your package.json file.

The difference between the two is dependencies are required to use the app, whereas devDependencies are production modules. They are not required to run the app, but rather for testing javascript, coffeescript, etc.

Why Use Prune?

Let say you have a big project. You will have a ton of modules you are working with and within those modules are MORE modules and even MORE modules. It's module inception! This is where prune comes in handy. Prune will remove any modules that are not included as dependencies in your package.json. Thereby saving you from having to remove them yourself!

How to use Prune

To use prune, you would use this syntax:

npm prune

npm prune modulename

npm prune modeulename -production (for devDependencies)

Example the First

So, we'll use phantomjs as an example. This folder is an extraneous package and isn't being used; there is no point to it. So we will use prune to get rid of it! You can do this by typing:

Npm prune phantomjs

Node will look at the file and realize we don't need it and get rid of the file!

Example the Second

Let's say you aren't sure what files are extraneous. You would just type the following:

Npm prune

Node will go through and check all of your files in the directory and search for extraneous files. Along with phantomjs, you will see that normalize-git-url, fake, and far have also been deleted.

Example the Third

In this final example, you will go inside one of phantomjs and go further indepth with prune.

If you go inside of phantomjs, you can also use prune here as well. But as of right now, prune won't do anything because all of the files are needed. But you will change that. Open the package.json, search for the dependencies list and remove one of the dependencies.

If you run prune now, it will show that a file has been unbuilt and removed.
